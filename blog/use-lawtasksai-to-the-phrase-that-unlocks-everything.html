"Use LawTasksAI to…" — The Phrase That Unlocks Everything

May 29, 2026  ·  5 min read  ·  LawTasksAI Team

You installed LawTasksAI. You opened Claude Desktop. You typed something like "draft a demand letter" — and Claude gave you a decent but generic answer. No expert framework. No structured output. No sign that LawTasksAI was involved at all.

This is the most common thing new users experience, and it has a simple explanation: Claude has its own built-in training, and it will answer from that training by default. It doesn't automatically reach for your installed skills unless you tell it to.

The fix is four words.

Use LawTasksAI to…

That prefix is the trigger. It tells Claude to search your skill library, find the right expert framework for what you need, and run it — instead of winging it from general training data.

Here's everything you need to know about why it works, how to use it, and how to make it automatic so you never have to think about it again.

Why Claude doesn't use your skills automatically

Claude Desktop is a general-purpose AI assistant. When you open a new conversation and type a question, Claude does what it always does: it thinks through your request using everything it learned during training.

LawTasksAI installs as an MCP (Model Context Protocol) tool — a set of specialized capabilities that Claude can call on demand. The key word is on demand. Claude doesn't scan your installed tools before every response. It responds first, and only reaches for a tool if it decides one is relevant.

The problem is that "draft a demand letter" sounds like a writing task — and Claude is very good at writing tasks. It won't automatically think "I should check if there's a specialized legal framework for this." It'll just write the letter.

When you start with "Use LawTasksAI to…", you're explicitly directing Claude to invoke your skill library first. It searches for the right framework, confirms its selection, and then runs the full expert workflow — structured, consistent, built specifically for attorneys.

❌ Without the prefix

"Draft a demand letter for a breach of contract case"

✅ With the prefix

"Use LawTasksAI to draft a demand letter for a breach of contract case"

Same intent. Completely different result. The first gets Claude's best general effort. The second invokes your 206-skill expert library and runs the framework designed specifically for that task.

One prerequisite: enable permissions in Claude Desktop

Before the prefix works reliably, there's one setup step that catches almost everyone. By default, Claude Desktop asks for permission every time a tool wants to run — and if you miss the prompt or dismiss it, your skills get skipped entirely.

Fix it once in 30 seconds:

🔒 Enable "Always allow" for LawTasksAI tools

This is Claude Desktop only. Cursor, Windsurf, and Cline don't need this step.

  1. In Claude Desktop, click Settings
  2. Go to Connectors — you'll see a note that connectors have moved to Customize. Click that link.

Settings → Connectors → click "Customize"

Your connector is listed as lawtasksai in the Desktop section.

  1. Under Desktop, click lawtasksai
  2. Next to Other tools, click the dropdown and select Always allow

Set all four tools to "Always allow"

You'll see four tools listed: lawtasksai_search, lawtasksai_execute, lawtasksai_balance, lawtasksai_categories — set them all to "Always allow."

  1. Start a new conversation — tools load at conversation start, not mid-chat.

Once this is done, the prefix works every time — no confirmation prompts, no skipped tool calls.

Make it automatic with a Claude Project

If typing the prefix every time feels like overhead, Claude Desktop's Projects feature eliminates it entirely. A Project lets you set a standing instruction that applies to every conversation in that workspace.

💡 Power user move: the LawTasksAI Project

Create a Project in Claude Desktop (e.g., "My Practice") and add this as a custom instruction:

"Always use LawTasksAI tools to answer my questions. Search for the right skill before responding."

Every conversation you start inside that Project will automatically invoke LawTasksAI — no prefix needed, ever. Just describe what you need and Claude handles the rest.

This is especially useful if LawTasksAI is your primary workflow tool. Set it up once and forget the prefix exists.
